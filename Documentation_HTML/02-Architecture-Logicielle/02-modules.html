02 — Modules logiciels

Inventaire des fichiers sources applicatifs et rôle de chaque module

📂 Périmètre de cette page

Seuls les modules présents dans Appli/Core/Src et Appli/Core/Inc sont détaillés. Les bibliothèques externes (HAL, BSP, AI Runtime, Camera Middleware, ScreenL, Tracker) sont décrites au niveau de leur API d'usage dans les pages suivantes.

2.1 Fichiers sources (.c)

Le code applicatif est organisé en 11 fichiers C. Les trois plus importants — main.c, app.c et app_cam.c — concentrent la logique métier ; les autres remplissent un rôle d'infrastructure (IRQ, MSP HAL, sécurité, runtime C).

FichierRôle principalComposant
main.c Point d'entrée bare-metal : init HAL, caches I/D, horloges système, RAM NPU, sécurité TrustZone, lancement du kernel μT-Kernel. Cœur
app.c Pipeline applicatif : crée les tâches RTOS, gère les buffer queues NN, exécute la boucle d'inférence, post-traitement, tracking et affichage. Cœur
app_cam.c Initialisation de la caméra via le middleware CMW, configuration des deux pipes DCMIPP (display + NN), mise à jour ISP périodique. Cœur
mtkernel_bsp.c Adaptation μT-Kernel ↔ STM32Cube (timer système, priorités IRQ). RTOS
app_fuseprogramming.c Programmation des fusibles de sécurité au premier démarrage. Sécurité
secure_nsc.c Stubs Non-Secure Callable exposés vers le monde non-sécurisé (TrustZone). Sécurité
stm32n6xx_hal_msp.c Callbacks HAL de configuration des broches et horloges des périphériques. HAL
stm32n6xx_it.c Vecteurs d'interruption STM32 (handlers IRQ). HAL
stm32_lcd_ex.c Extensions de l'API LCD STM32 (formatage texte, primitives graphiques). Affichage
utils.c Utilitaires : mesure de la charge CPU, profilage temporel. Outil
syscalls.c · sysmem.c Stubs C standard (newlib) : _sbrk, _write… Runtime C

En-têteRôle

cmw_camera_conf.h · isp_conf.h · isp_param_conf.hConfiguration des capteurs caméra et de l'ISP.
imx335_isp_param_conf.h · vd66gy_isp_param_conf.hProfils ISP par capteur.
aps256xx_conf.h · mx25um51245g_conf.h · mx66uw1g45g_conf.hConfigurations des mémoires XSPI (RAM externe et flash NOR).
partition_stm32n657xx.hPartition mémoire TrustZone (régions secure / non-secure).
stm32n6xx_hal_conf.h · stm32_assert.hConfiguration HAL et macro assert_param.

2.3 Schéma de dépendances

main.c
  ├── HAL / BSP STM32Cube_FW_N6
  ├── mtkernel_bsp.c → μT-Kernel 3.0
  └── app.c
        ├── app_cam.c → CMW Camera Middleware → capteurs IMX335 / VD55G1 / VD66GY
        ├── network.h (NeuralART AI Runtime) → Modèle YOLO-X
        ├── postprocess_conf.h → ai-postprocessing-wrapper → lib_vision_models_pp
        ├── tracker (TRACKER_MODULE)
        ├── ScreenL (SCRL) → LTDC
        └── serial_protocol.h → tm_printf (μT-Kernel)
