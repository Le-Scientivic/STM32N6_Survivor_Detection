05 — Protocole série JSON

Format des messages émis sur USART1 — intégration avec des systèmes externes

📡 Liaison de sortie

Le firmware émet des événements structurés sur la console USART1 (115200 baud, 8N1). Le format est défini dans Appli/Core/Inc/serial_protocol.h et est sélectionnable parmi FORMAT_JSON (par défaut), FORMAT_DEBUG ou FORMAT_BOTH.

5.1 Format général d'un message

[TIMESTAMP] [LEVEL] [MODULE] [EVENT] {json_data}
ChampDescription
TIMESTAMPMillisecondes depuis le boot, sur 8 caractères, padding zéros
LEVELNiveau : INFO · DATA · PERF · WARN · ERROR · DEBUG
MODULESource du message (cf. table ci-dessous)
EVENTType d'événement (cf. table ci-dessous)
json_dataPayload JSON spécifique à l'événement

5.2 Modules

9 modules sources sont définis dans serial_protocol.h :

ModuleÉmetteur typique
INITPhase d'initialisation système
RTOSCréation / démarrage des tâches μT-Kernel
CAMERADriver caméra et ISP
NNTâche d'inférence
PPPost-traitement YOLO
DETECTRésultats finaux de détection
TRACKSuivi des objets (IDs persistants)
DISPLAYTâche d'affichage
PERFMesures de performance

5.4 Événement de détection

À chaque frame, le module DETECT émet un message DETECTION_RESULT contenant les boîtes englobantes :

Sans tracking

[00012345] [DATA] [DETECT] [DETECTION_RESULT]
{
  "frame": 150,
  "count": 2,
  "tracking": false,
  "objects": [
    { "class": "person", "bbox": { "x": 120, "y": 85, "w": 45, "h": 120 } },
    { "class": "person", "bbox": { "x": 450, "y": 90, "w": 50, "h": 130 } }
  ]
}

Avec tracking activé

[00023456] [DATA] [DETECT] [DETECTION_RESULT]
{
  "frame": 250,
  "count": 3,
  "tracking": true,
  "objects": [
    { "id": 1, "class": "person", "bbox": { "x": 115, "y": 82, "w": 48, "h": 125 } },
    { "id": 3, "class": "person", "bbox": { "x": 455, "y": 88, "w": 52, "h": 135 } },
    { "id": 7, "class": "person", "bbox": { "x": 200, "y": 150, "w": 40, "h": 110 } }
  ]
}
ChampTypeDescription
frameintegerNuméro de frame (compteur croissant)
countintegerNombre de survivants détectés
trackingbooleanIndique si le tracker est actif
objects[].idintegerID persistant (tracking activé uniquement)

5.5 Événements de performance

Le module PERF publie un message TIMING par frame :

[00012389] [PERF] [PERF] [TIMING]
{
  "frame": 150,
  "inference_ms": 32,
  "pp_ms": 4,
  "display_ms": 8,
  "period_ms": 44,
  "fps": 22
}

Le module DISPLAY émet un FRAME_START à l'entrée du rendu :

[00012345] [DATA] [DISPLAY] [FRAME_START]
{ "frame": 150, "count": 2, "tracking": false, "buffer": 0 }

5.6 Avertissements

Lorsque le post-processing détecte plus d'objets que la limite (AI_OD_ST_YOLOX_PP_MAX_BOXES_LIMIT = 10), le module PP émet :

[00034567] [WARN] [PP] [WARNING]
{ "msg": "detection_overflow", "detected": 15, "limit": 10 }

5.8 Exemple de parser Python

import re, json, serial

def parse(line):
    m = re.match(r'\[(\d+)\] \[(\w+)\] \[(\w+)\] \[(\w+)\] (.+)', line)
    if not m: return None
    ts, lvl, mod, evt, data = m.groups()
    return { 'ts': int(ts), 'level': lvl, 'module': mod, 'event': evt,
             'data': json.loads(data) }

port = serial.Serial('/dev/ttyUSB0', 115200, timeout=1)
while True:
    line = port.readline().decode('utf-8').strip()
    msg = parse(line)
    if msg and msg['module'] == 'DETECT':
        print(f"Frame {msg['data']['frame']} : {msg['data']['count']} survivant(s)")
