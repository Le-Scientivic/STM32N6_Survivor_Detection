01 — Vue d'ensemble & Environnement

Plateforme logicielle, séquence de boot, organisation du code et environnement de développement

🎯 Mission du firmware

Le projet STM32N6 Survivor Detection est un firmware embarqué temps-réel qui détecte la présence de personnes (survivants) dans le flux d'une caméra MIPI CSI-2, en exploitant l'unité NPU intégrée à la puce STM32N657 pour exécuter un réseau de neurones YOLO-X Nano à 25–30 images par seconde, avec affichage live des boîtes englobantes sur l'écran LCD intégré.

1.1 Plateforme logicielle

Le firmware est construit autour de quatre couches superposées :

CoucheComposantRôle
ApplicationAppli/Core/Pipeline métier de détection (acquisition, inférence, post-traitement, affichage)
RTOSμT-Kernel 3.0 BSP2 (T-License 2.2)Multitâche préemptif, sémaphores, mutex, files de buffers
MiddlewaresCMW Camera · NeuralART AI Runtime · Tracker · ScreenL · YOLO post-processingAbstraction des périphériques et services techniques
HAL / BSPSTM32N6xx HAL Driver · BSP STM32N6570-DKAccès aux périphériques (DCMIPP, LTDC, XSPI, USART, NPU…)

1.2 Séquence de boot en deux temps

Conformément à l'architecture sécurisée du STM32N6, le démarrage se déroule en deux phases :

Power-On
   │
   ▼
┌─────────────────────────────────────────┐
│  FSBL (First Stage Boot Loader)         │
│  Exécution dans AXISRAM2                │
│  → Init horloges, sécurité, RIF/IAC     │
│  → Chargement de l'application en XSPI  │
│  → Saut vers main()                     │
└─────────────────────┬───────────────────┘
                      ▼
┌─────────────────────────────────────────┐
│  Application STM32N6_MTK_Person_*       │
│  Exécution depuis XSPI RAM (XIP)        │
│  → main() : init HAL, NPU, μT-Kernel    │
│  → usermain() : entry RTOS              │
│  → main_thread_fct() : init bus + app   │
│  → app_run() : pipeline détection       │
└─────────────────────────────────────────┘

1.3 Points d'entrée du firmware

FonctionFichierRôle
main()Appli/Core/Src/main.cInitialisation HAL, caches, horloges, NPU, sécurité ; lance le kernel
usermain()main.cEntrée utilisateur du μT-Kernel : crée la tâche principale
main_thread_fct()main.cInitialise les bus XSPI, configure les priorités NVIC, appelle app_run()
app_run()Appli/Core/Src/app.cCrée les 4 tâches du pipeline, initialise écran et caméra, boucle infinie

1.6 Environnement de développement

  • IDE : STM32CubeIDE 1.19.0 ou supérieur (toolchain ARM GCC inclus)
  • Programmateur : STM32CubeProgrammer 2.20.0 ou supérieur
  • Cible matérielle : STM32N6570-DK Discovery Kit avec ST-LINK on-board
  • Configuration matérielle : fichier STM32N6_MTK_Person_Detection.ioc (STM32CubeMX)
  • Génération du modèle : Model/STM32N6570-DK/generate-n6-model.sh
