6.2 Configuration de compilation

Le projet est compilé en optimisation taille (-Os) avec les options suivantes :

-mcpu=cortex-m55
-mthumb
-mfpu=fpv5-d16
-mfloat-abi=hard
-mcmse                # Activation TrustZone (Secure)
-Os

6.4 Procédure de build dans STM32CubeIDE

  1. Ouvrir STM32CubeIDE.
  2. File → Import → Existing Projects into Workspace et pointer la racine du dépôt.
  3. Importer le projet STM32N6_MTK_Person_Detection.
  4. Clic droit sur Appli → C/C++ Build → Settings → Build Steps.
  5. Adapter la commande post-build avec le chemin réel vers STM32_SigningTool_CLI (situé dans STM32CubeProgrammer/bin).
  6. Clic droit sur Appli → Build Project.
  7. Vérifier la production de Appli/Debug/STM32N6_MTK_Person_Detection_Appli-Trusted.bin.

6.5 Programmation des binaires (mode Dev Boot)

⚠️ Mode Dev Boot

Avant la programmation, configurer la carte en BOOT0 = LOW · BOOT1 = HIGH.

Trois binaires doivent être programmés dans l'ordre suivant via STM32CubeProgrammer :

OrdreFichierAdresse / Action
1 Binaries/ai_fsbl.hex Hex auto-routé (FSBL)
2 Binaries/network_data-dk.hex (poids du modèle) Hex auto-routé (NOR flash)
3 Binaries/STM32N6_MTK_Person_Detection_Appli-Trusted.bin Adresse 0x70100000

6.6 Lancement de l'application (mode Flash Boot)

  1. Déconnecter STM32CubeProgrammer.
  2. Repasser la carte en BOOT0 = LOW · BOOT1 = LOW (Flash Boot).
  3. Reset (ou cycle d'alimentation).
  4. Le FSBL démarre, charge l'application en RAM externe et l'exécute.
  5. Surveiller la sortie sur USART1 via un terminal série (115200 baud).
